# 上游映射与泛化说明

这个仓库不是把上游 `Product Manager Skills` 原样翻译成中文，而是做了“面向商业产品经营”的重组。

## 映射原则

1. 保留上游里真正可复用的分析骨架
2. 去掉过度 SaaS 化、过度通用化的部分
3. 增加商业产品团队更常用的经营口径
4. 输出从框架改成“结论 + 动作 + 风险 + 验证”

## 主要映射

| 上游能力 | 本仓库承接能力 | 泛化方式 |
| --- | --- | --- |
| `business-health-diagnostic` | `revenue-health-diagnostic` | 从经营健康度扩展到收入结构、漏斗、留存、渠道效率和单位经济 |
| `finance-based-pricing-advisor` | `pricing-change-evaluator` | 从 SaaS 定价评估扩展到订阅、电商、平台抽佣、增值包和促销设计 |
| `feature-investment-advisor` | `feature-investment-evaluator` | 从功能投入建议升级为收益路径、成本结构和实验路径评估 |
| `acquisition-channel-advisor` | `acquisition-channel-evaluator` | 更强调后链路质量、回本和规模化，而不是只看前链路获客 |
| `prioritization-advisor` | `commercial-prioritization-advisor` | 从通用优先级框架转成商业化需求排序逻辑 |
| `product-strategy-session` | `commercial-growth-strategy-workflow` | 从产品策略讨论升级为经营诊断到路线图的可执行流程 |
| `jobs-to-be-done` + `problem-framing-canvas` + `tam-sam-som-calculator` | `commercial-opportunity-canvas` | 把用户需求、机会大小和变现路径合成一个商业机会画布 |
| 上游缺少完整续费/召回闭环 | `retention-revenue-playbook` | 新增商业产品常见的续费、复购、召回、挽回和 dunning 策略能力 |

## 当前仓库相对上游新增的视角

- 从“用户价值”补到“收入质量”
- 从“功能优先级”补到“经营动作优先级”
- 从“提出方案”补到“分层、测算、灰度和验证”
- 从“PM 单点产出”补到“商业产品团队协作”

## 后续可继续扩展的方向

- 分国家和支付方式的定价策略
- 广告变现专项技能
- Marketplace 双边效率专项技能
- 商业化实验设计和归因专项技能
