# Subscription Renewal Diagnostic

## Purpose

这个 skill 用于诊断会员 / 订阅业务的续费问题，不是泛泛看留存，也不是只看一个总体续费率。

它默认覆盖：

- 在期续费
- 已解约但未到期
- 过期回流
- 支付失败恢复

## Key Concepts

### 1. 先拆口径，再看结论

至少分开以下几类：

- 试用转付费
- 付费用户到期续费
- 已取消自动续费用户的挽回
- 过期后的召回
- 支付失败后的恢复

### 2. 续费问题通常发生在 4 个环节

1. 价值感没建立
2. 临期提醒和权益表达弱
3. 解约后没有有效挽回
4. 过期后召回或支付恢复薄弱

### 3. 一定要分层

最低拆分：

- 国家 / 地区
- SKU / 周期
- 生命周期状态

再补：

- 渠道
- 新老用户
- 高价值 / 低价值

## Required Inputs

- 业务区域与 SKU 结构
- 各层级续费、解约、过期回流数据
- T-7 / T-3 / T-1 与 D1 / D7 / D14 / D30 触达表现
- 支付失败率与恢复率
- 当前主要策略与最近变更

## Application

### Step 1: 明确目标口径

先说明你要优化的是：

- 首次续费
- 多次续费
- 取消挽回
- 过期召回
- 支付失败恢复

### Step 2: 搭建诊断切面

推荐至少形成以下切面：

- 地区 × SKU
- 生命周期状态 × 续费率
- 触达窗口 × 转化

### Step 3: 找关键流失环节

输出时必须指出：

- 最大掉点在哪个生命周期阶段
- 是策略问题、价格问题、权益问题还是支付问题
- 哪一类人群最值得优先干预

### Step 4: 形成动作方向

动作方向通常落在：

- 调整临期触达节奏
- 加强取消挽回
- 做短过期召回
- 做支付失败恢复

## Output Format

1. 一句话续费健康度结论
2. 分层诊断表
3. 最大问题环节
4. 高优先级人群
5. 30 天修复动作
6. 待补数据

## Examples

### 示例：海外周卡续费差

结论不应只写“海外续费差”，而应写成：

- 海外周卡在 T-3 到期提醒后转化弱
- 已取消自动续费用户占比高
- 支付失败率正常，因此优先级不在支付恢复

## Common Pitfalls

- 把试用转付费当续费
- 只看一个整体续费率
- 不区分已解约用户和未解约用户
- 把短过期召回和长过期召回混着看
