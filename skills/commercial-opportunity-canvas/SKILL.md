# Commercial Opportunity Canvas

## Purpose

这个 skill 用于判断一个商业机会是否值得进入下一步，而不是直接帮你写方案。

它特别适合以下情况：

- 业务方提出了一个看起来有收入空间的新方向
- 经营会上出现多个机会，但缺少统一比较口径
- 你需要先说清楚“为什么是这个机会、为什么是现在、为什么是这类用户”

## Key Concepts

### 1. 机会不是需求点子

商业机会至少要同时回答 5 个问题：

1. 谁会为这件事产生价值感知
2. 价值发生在什么使用场景
3. 价值能否转化为收入或利润
4. 规模是否足够支撑投入
5. 有没有现实的上线与验证路径

### 2. 需求强度和付费强度不是一回事

用户经常会表达需求，但不一定愿意付钱。要区分：

- 高频强需求，但付费意愿弱
- 低频刚需，但单次价值高
- 用户喜欢，但公司难赚钱
- 公司能赚钱，但用户难感知价值

### 3. 先看变现路径，再谈功能

机会画布优先梳理：

- 收入来自哪里
- 谁付钱
- 是一次性收入、持续收入还是间接收入
- 变现动作发生在主链路哪个环节

### 4. 机会评估必须包含约束

再大的空间，如果受制于以下约束，也不应该直接推进：

- 支付能力不够
- 供给或履约能力不够
- 毛利过低
- 合规风险高
- 当前组织资源无法承接

## Application

### Step 1: 用一句话定义机会

推荐格式：

`针对 [目标人群]，在 [具体场景] 中，通过 [价值机制] 创造 [收入或利润结果]。`

### Step 2: 定义目标人群和触发场景

至少说明：

- 主要人群是谁
- 这个机会发生在什么关键时刻
- 当前替代方案是什么
- 用户为什么现在会考虑付费或切换

### Step 3: 验证价值与支付意愿

至少给出下面 3 类证据中的 2 类：

- 用户反馈或访谈
- 行为数据或漏斗异常
- 市场对标或竞品收费事实

### Step 4: 明确变现路径

至少写清楚：

- 收费对象
- 收费方式
- 关键收入指标
- 可能的成本结构

### Step 5: 初步估算空间与门槛

不要求精确 TAM，但至少回答：

- 机会覆盖多大用户盘
- 单用户价值大概多少
- 上线后最早多久能看到结果
- 最大的不确定性在哪

### Step 6: 给出推进建议

结论只能是以下四类之一：

- 立即进入方案设计
- 先做轻量验证
- 作为机会池保留
- 当前不做

## Examples

### 示例：工具型会员加推“批量处理权益包”

1. 目标人群：有重复处理需求的中高频创作者
2. 场景：单次处理量上升，现有基础会员权益不够
3. 价值机制：节省时间并提高产出效率
4. 变现路径：在会员体系上叠加高阶权益包
5. 风险：重度用户规模可能比预期小，需要先验证付费转化
6. 推荐动作：先对高频用户做权益意愿调研和小流量售卖页验证

## Common Pitfalls

- 只有“市场很大”，没有目标人群和触发场景
- 只有用户需求，没有收入路径
- 只看收入，不看毛利和履约成本
- 直接把机会当成功能方案，跳过验证
