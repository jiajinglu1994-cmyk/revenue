# Pricing Change Evaluator

## Purpose

对一个已经被提出的定价变化做 go / no-go 评估。重点不是“理论上怎么定价”，而是判断这个变化是否值得上线、先灰度还是暂缓。

## Key Concepts

### 1. 定价变化不只影响收入

至少同时影响：

- 付费转化
- 续费或复购
- 退款和投诉
- 套餐迁移
- 毛利和回本

### 2. 一定要做分层测算

平均值最容易掩盖问题。优先拆：

- 新用户 vs. 老用户
- 高价值用户 vs. 低价值用户
- 不同价格带
- 不同地区与渠道

### 3. 优先判断变化类型

变化类型不同，风险重点不同：

- 直接涨价：看弹性和舆情风险
- 折扣变化：看转化提升是否透支利润
- 套餐重构：看迁移和互相蚕食
- 周期迁移：看现金流与长期续费
- 抽佣变化：看供需两端行为变化

### 4. 定价结论必须带灰度方案

商业产品的定价很少适合全量一次性上线，优先考虑：

- 新用户先行
- 特定市场或渠道先行
- 高价值人群先试
- 限时灰度观察退款和投诉

## Typical Scenarios

- 年卡涨价或降价
- 新增高阶套餐
- 折扣策略调整
- 试用转付费门槛变化
- 抽佣比例变化
- 增值包、附加包、权益包单独售卖

## Required Inputs

- 当前价格体系
- 拟议变化内容
- 受影响人群
- 当前转化率、续费率、退款率或流失率
- ARPU、ARPPU、客单价或抽佣收入
- 毛利率和历史活动效果

## Application

### Step 1: 先定义变化类型

把变化归到一种主类型：

1. 直接涨价或降价
2. 套餐拆分或打包
3. 周期变化，例如月付转年付
4. 促销折扣变化
5. 平台费率或抽佣变化
6. 增值包/附加包新增收费

### Step 2: 判断影响路径

每次定价变化至少要分析以下 5 条路径中的 3 条：

- 收入提升
- 转化损失
- 流失或退款风险
- 套餐迁移和互相蚕食
- 毛利与回本变化

### Step 3: 做三档情景测算

至少输出保守、基准、乐观三档。

建议统一回答：

- 收入变化幅度
- 利润或毛利变化幅度
- 用户规模变化幅度
- 回本周期变化

### Step 4: 看分层，不看平均值

必须分层看受影响人群，例如：

- 新用户 vs. 老用户
- 低价带 vs. 高价带
- 重度用户 vs. 轻度用户
- 国内 vs. 海外
- 不同渠道来源

### Step 5: 形成上线建议

建议只能落到下面四类之一：

- 直接上线
- 小流量灰度
- 仅针对某一人群上线
- 暂缓，先补数据或先做实验

## Output Format

1. 定价变化摘要
2. 影响路径说明
3. 三档测算表
4. 风险点
5. 推荐动作
6. 灰度方案
7. 需要继续验证的数据

## Examples

### 示例：年卡涨价评估

假设：

- 当前年卡价格 198 元
- 拟调到 228 元
- 新用户为主，老用户先不动

输出时至少要回答：

1. 涨价后基准情景的收入提升是多少
2. 允许承受的转化损失上限是多少
3. 是否建议先在单一渠道灰度
4. 如果投诉和退款超阈值，何时回滚

## Common Pitfalls

- 只算涨价收入，不算转化和退款损失
- 只看整体付费率，不看分层弹性
- 忽略对老用户体验和舆情的冲击
- 忽略套餐互相蚕食
